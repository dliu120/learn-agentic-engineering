# ALLM Academy

An immersive, **self-updating** static course on **production-LLM engineering** for AI engineers —
the systems work *around* the model. Nine modules cover 22 hard topics (harness design, inference
internals, model efficiency, structured output & tools, agent control, RAG, evals & observability,
cost/safety/multi-tenancy, and a strategy capstone), taught with animated 2D diagrams, interactive
quizzes, progress tracking, and optional build-time audio narration.

A daily CI job curates AI news (Hacker News, arXiv, OpenAI/HF/blog RSS) into short teachable
briefings mapped back to the curriculum, so the site stays current without a runtime backend.

> Built from a spec in [`goal-prompt.md`](./goal-prompt.md). Tech: Astro 4 (static) · Preact islands ·
> Tailwind · GSAP-free SVG/Canvas viz · Pagefind search · edge-tts audio.

## Quick start

```bash
npm install
npm run dev            # http://localhost:4321
npm run build          # static output in dist/ (astro build + pagefind index)
npm run preview        # serve the production build
```

```

    config.ts                 Zod collections: lessons (MDX), daily (JSON), glossary (MD)

    modules.ts                9-module taxonomy (order, hue, objectives, keywords)
    lessons/<moduleId>/*.mdx  22 topic lessons + 9 module-gate quizzes
    daily/<date>.json         generated briefings (+ 2 example fixtures)
    glossary/*.md
  components/
    islands/   Quiz, AudioPlayer, CommandPalette (Preact, hydrated client-side)
    viz/       11 animated 2D diagrams (one hero per module) + lazy-loading wrapper + stepper
    ui/        ConceptCard, KeyInsightCallout, MisconceptionFlip, FailureModeList, TradeoffTable,
               ModuleCard, DailyLessonCard, CurationFlagBadge, Header/Footer …
  layouts/     BaseLayout (theme resolution, ⌘K palette), Dashboard/Lesson chrome in pages
  lib/         progress.ts (localStorage engine), manifest.ts, paths.ts, motion.ts, daily.ts
  pages/       dashboard, modules, lessons, today, daily/[date], archive, calendar, glossary,
               search, about, feed.xml

               · curate.ts (guarded LLM + repair) · run.ts (orchestrator)
  gen-audio.ts · check-coverage.ts · lib/{http,llm,log,frontmatter}.ts
```

### Design decisions
- **Static only.** No runtime server. The daily feed is regenerated by CI and committed.
- **Reduced-motion first.** Every viz renders a correct static final state and exposes a text
  equivalent; nothing depends on motion.
- **Progress is local.** `localStorage` only — export/import on the About page. No accounts, no tracking.
- **Quiz answer encoding:** mcq `correct:number`, multi/ordering `correct:number[]`, matching uses
  `pairs[]` as the canonical mapping (no `correct`).

## The daily pipeline

`npm run daily` fetches public feeds (see `sources.yaml`), dedupes/filters/ranks by relevance +
recency, then curates the top items into lessons mapped to the 9 modules:

- **With** `ANTHROPIC_API_KEY` → LLM curation (summary + why-it-matters + micro-quiz), validated by
  Zod with a bounded repair loop. Flag: `curated`.
- **Without** a key → deterministic templated summaries. Flag: `uncurated-fallback`.
- **No candidates** clear the bar → a `quiet-day` evergreen card.

All failure paths are graceful (a dead feed, bad LLM output, or missing edge-tts never crashes the
run). Copy `.env.example` → `.env` to configure.

### Honest limits
- Lesson numbers/prices/benchmarks are illustrative ranges — verify against primary sources.
- Reddit often returns 403 to automated requests and is skipped; Anthropic has no stable public RSS
  (proxied via Google News). X/Twitter has no keyless feed and is intentionally omitted.

## Deploy

Two GitHub Actions workflows are included:

- **`.github/workflows/deploy.yml`** — builds and deploys to **GitHub Pages** on push to `main`
  (needs only `GITHUB_TOKEN`). For a project site set `SITE`/`BASE` (see the workflow + `.env.example`).
- **`.github/workflows/daily.yml`** — cron job that runs the pipeline, commits the new day, and
  triggers a redeploy. Add `ANTHROPIC_API_KEY` as an Actions secret to enable LLM curation.

Any static host works (Netlify/Vercel-static/Cloudflare Pages) — just serve `dist/`.

## Definition of Done status

See `goal-prompt.md` §15. Met: clean `build` + `astro check` (0 errors); all 22 topics
(`npm run check:coverage`); per-module animated diagram + quiz + gate; mcq/multi/ordering/matching;
localStorage progress + export/import; guarded edge-tts audio + transcripts; daily pipeline
(dry-run + live) with graceful degradation; today/archive/calendar/news/feed; Pagefind search + ⌘K;
dark mode; reduced-motion. Lighthouse tuning and full edge-tts audio generation are environment-gated
(run `gen:audio` with edge-tts installed; run Lighthouse against `dist/`).
